GiveBlack

Operations & app guide

This page is for your team: how the GiveBlack mobile app works for donors and charities, and how to use the admin backoffice. It is marked noindex and not intended for public search results.

URL: you may be viewing this at /app/adminguide/ (API prefix). After nginx is updated with a top-level /adminguide block, the same content will also answer at /adminguide/ on the main domain.

Mobile app (Expo)

The consumer app uses tab-based navigation and role-specific flows. Paths below are logical screens (Expo Router), not URLs in a browser.

Get started

Donors: create an account (email/password or social where enabled), sign in, complete onboarding if shown.

Charities: use charity signup to submit organization details; approval may be required before full dashboard access.

Auth screens include donor login/signup, charity login/signup, password reset, and welcome flows.

Donor experience

  • Home tab — discovery feed and entry points to causes.
  • Give tab — browse ways to give and act on campaigns.
  • Categories — filter or explore by category.
  • Favourites — saved organizations or campaigns.
  • Account — profile, settings, and sign-out.
  • Search — find organizations or campaigns.
  • Donate — org-scoped donation flow (donate/[orgId]).
  • Campaign — view a campaign and donate (campaign/[id]).
  • Volunteer — sign up to volunteer with an organization (volunteer/[orgId]).
  • Community — browse and open community campaigns; create flow where enabled.
  • Impact — see giving impact and history-oriented summaries.
  • Notifications — in-app notification list.
  • Checkout result — return state after payment flows.
  • Top-up / wallet-style flows — where configured in the build.

Charity / organization dashboard

When signed in as a charity owner, the app exposes organization tools, typically including:

  • Organization home, profile, and settings.
  • Campaigns list and management.
  • Donations and payment history for the org.
  • Volunteer signups for the org.
  • Payouts / Stripe Connect–related views where enabled.
  • Subscriptions (org subscription tier).
  • Plan / billing and fee-related settings where exposed in app.

Admin backoffice

Operators use the web backoffice to manage users, organizations, campaigns, money movement, and platform settings. Your deployment should expose it at a path like /backoffice/ on the same host as the public site.

URL & sign-in

Open the backoffice: BACKOFFICE_URL_PLACEHOLDER

Sign-in methods:

  • Email one-time code — enter your authorized staff email, receive a short code, enter it to complete login.
  • Google — use the Google button where configured; your Google account must match an authorized user record.

Password login is disabled; use OTP or Google as shown on the login screen.

Staff management: only admin and super_admin can add, edit, or remove staff and change roles. You can grant an existing donor email staff access; the account is upgraded in place (same email, new role).

Permission overrides on staff/manager rows can gate actions such as managing users, changing roles, platform settings, and payments — enforced on the API. If an action is denied, ask a super admin to adjust overrides or role.

What each backoffice screen is for

Dashboard — high-level stats and shortcuts; start here each session.

Users — search and inspect donor/user records; open a donor detail view by email where linked.

Organizations — list charities/orgs; open an org to edit profile, Stripe/Connect state, campaigns, media, and related tools.

Campaigns — platform-wide campaign list and detail editing for org campaigns.

Community — community campaigns (separate from org-only campaigns); create/edit/moderate as your policy allows.

Charity requests — review pending charity applications; approve or reject with notes.

Volunteers — review volunteer signups across organizations.

Donations — payment records, statuses, and reconciliation aids.

Subscriptions — organization subscription tiers and billing hooks (Stripe).

Transactions — internal transaction listing for finance review.

Fund release — workflows to release held funds where your process uses this tool.

Ledger — ledger-style entries for auditing money movement.

GiveBlack financials — aggregate platform financial views for leadership.

Categories — taxonomy for organizations and filters in the app.

Education partners — partners used for education/reinvest attribution features.

Staff — maintain who can access the backoffice and with which role; set permission overrides per person.

Admin emails — list of addresses that receive operational emails; send a test message to verify delivery.

Notify users — broadcast push/in-app notifications to selected donor or charity segments (admin-gated).

Settings — platform-wide configuration keys exposed to the admin UI.
