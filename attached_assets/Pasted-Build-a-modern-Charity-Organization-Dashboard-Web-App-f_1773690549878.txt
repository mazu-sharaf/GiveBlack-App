Build a **modern Charity / Organization Dashboard Web App** for a donation platform.
The **donor interface and donation flow already exist**, so this project must focus only on the **charity / organization side** including authentication, campaign management, donations analytics, and subscription-based limits.

The design must be **premium, modern, clean, and responsive**, similar to **Stripe Dashboard, Notion, and Linear UI quality**.

Use a **tabbed application layout with bottom navigation (mobile) and sidebar navigation (desktop)**.

Tech stack preference:
React + Tailwind + Vite
Firebase or Supabase for auth and database
Stripe for subscriptions and billing

---

## 1. Authentication and Organization Approval

Create an authentication system specifically for **charity organizations**.

Signup fields:
Organization Name
Category
Organization Description
Website
Contact Person Name
Email
Password
Optional fields:
Organization Address
Phone Number
Registration Number / Legal ID

Signup flow:

When an organization signs up, store their account with status **pending**.

Admin must review the request and either **approve or reject**.

When approved:
Create an **Organization record**
Link the user account to that organization.

Only **approved organizations can access the dashboard**.

If the account is still pending show:

"Your organization request is under review. You will receive an email once approved."

## 3. Subscription Tiers

Organizations operate under **subscription plans**.

Create three tiers:

Free Plan

Price: $0
Campaign Limit: 2
Maximum Goal per Campaign: $5,000
Basic analytics

Growth Plan

Price: $29/month
Campaign Limit: 10
Maximum Goal per Campaign: $50,000
Advanced analytics
Campaign updates
Priority support

Institutional Plan

Price: $99/month
Unlimited campaigns
Unlimited campaign goals
Advanced analytics
Team members
Priority support

Store subscription data in the organization record.

Important limits:

Campaign creation must stop when limit reached.
Campaign goal cannot exceed plan max goal.

Payments handled through **Stripe Checkout**.

Billing management handled through **Stripe Customer Portal**.
